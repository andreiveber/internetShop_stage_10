# Интернет-магазин (учебный проект) - Этап 6

## Описание проекта

На данном этапе в проекте организовано хранение данных с помощью моделей и моделей представлений (ViewModels)

## Функциональность

В проекте реализовано:
- контроллер `HomeController` с внедрением зависимости `ProductService`;
- модель `Product` для хранения данных о товарах;
- модели представлений `HomeViewModel`, `ProductViewModel`, `ContactViewModel`;
- Cервис `ProductService` для централизованного управления данными;
- макет `_Layout.cshtml` для всех страниц с навигацией и фоновым изображением;
- область Admin с отдельным контроллером и представлениями;
- шесть страниц страниц (представления):
  - **главная страница (Index)** — приветствие и описание магазина с данными из ViewModel;
  - **страница с товарами (Products)** — динамический список товаров из сервиса с возможностью поиска по названию;
  - **страница контактов (Contact)** — адрес, телефон, email, часы работы с данными из ViewModel;
  - **страница деталей товара (ProductDetails)** — полная информация о товаре по ID;
  - **страница "О нас" (About)** — информация о компании;
  - **панель администратора (Admin/Dashboard)** — статистика и управление.

## Модели данных
- `Product` — основная модель товара (Id, Name, Price, Description).

## Модели представлений (ViewModels)
- `ProductViewModel` — расширенная модель товара для отображения (добавлены поля Category, InStock, ImageUrl);
- `HomeViewModel` — модель для главной страницы (WelcomeMessage, FeaturedProducts, TotalProducts, SpecialOffer);
- `ContactViewModel` — модель для страницы контактов (Address, Phone, Email, WorkingHours, UserName).

##  Атрибуты маршрутов в контроллерах
- `[Route("catalog")]` — псевдоним для страницы товаров;
- `[Route("product/{id:int}")]` — детальная страница с параметром; 
- `[Route("feedback/{name?}")]` — опциональный параметр в URL;
- `[Route("about")]` — страница "О компании";
- `[Area("Admin")]` и `[Route("admin/[controller]")]` — настройка области администратора.
